# Plan Review Policy

Example policy document that gets inlined into the bug-plan template via
`{{doc:docs/plan-review-policy.md}}`. Replace this content with your
team's actual review checklist — scope guardrails, risk categories,
test requirements, and whatever else a reviewer should weigh.

Keeping the policy in a separate file (instead of inline inside the
template) lets multiple templates reference the same policy, and makes
the policy diff cleanly in PRs when it changes.
